1. Project Overview

Cloud Coder is an AI coding assistant CLI tool that provides a unified interface for multiple LLM providers, with a focus on hybrid cloud/local inference through Ollama Cloud.


Key Capabilities:

  • Run 480B+ parameter models on Ollama Cloud while using local hardware for smaller tasks
  • One CLI across all backends: OpenAI, Gemini, GitHub Models, Codex, Ollama (local + cloud), Atomic Chat, Bedrock, Vertex, Foundry
  • Complete agent workflows: bash, file tools, grep, glob, agents, tasks, MCP, and web tools
  • VS Code extension for launch integration and theme support
  • Headless gRPC server for CI/CD integration

5. Tool System

Tool Interface (src/Tool.ts)

The Tool type defines the complete interface for all tools:

export type Tool<Input, Output, Progress> = {
  name: string
  aliases?: string[]
  searchHint?: string
  inputSchema: Zod schema
  outputSchema?: Zod schema

  // Core execution
  call(args, context, canUseTool, onProgress): Promise<ToolResult>
  description(input, options): Promise<string>

  // Permission & validation
  validateInput?(input, context): Promise<ValidationResult>
  checkPermissions(input, context): Promise<PermissionResult>
  isConcurrencySafe(input): boolean
  isReadOnly(input): boolean
  isDestructive?(input): boolean

  // UI rendering
  renderToolUseMessage(input, options): React.ReactNode
  renderToolResultMessage(output, options): React.ReactNode
  renderToolUseProgressMessage?(progress, options): React.ReactNode
  getActivityDescription?(input): string | null

  // Lifecycle
  isEnabled(): boolean
  interruptBehavior?(): 'cancel' | 'block'
}

MCP Tool Integration

MCP tools are discovered from connected servers and merged with built-in tools:

export function assembleToolPool(
  permissionContext: ToolPermissionContext,
  mcpTools: Tools,
): Tools {
  const builtInTools = getTools(permissionContext)
  const allowedMcpTools = filterToolsByDenyRules(mcpTools, permissionContext)

  // Deduplicate by name (built-in tools take precedence)
  return uniqBy(
    [...builtInTools].sort(byName).concat(allowedMcpTools.sort(byName)),
    'name',
  )
}

7. Permission System

Permission Modes

  • default - Ask for dangerous operations
  • auto - Auto-approve with classifier oversight
  • bypass_permissions - Skip permission prompts

Permission Rules

Rules are parsed from settings and applied hierarchically:

export type PermissionRule = {
  source: PermissionRuleSource  // settings, cliArg, command, session
  ruleBehavior: 'allow' | 'deny' | 'ask'
  ruleValue: {
    toolName: string
    subcommand?: string
  }
}

Example rules:

  • Bash(git *) - Allow all git commands
  • Read(src/*) - Allow reading src directory
  • Edit - Ask for all edit operations

Auto-Mode Classifier

The transcript classifier analyzes bash commands for security risks:

// src/utils/permissions/yoloClassifier.ts
export function classifyYoloAction(transcript: string): {
  action: 'approve' | 'reject' | 'ask'
  reason: string
  confidence: number
}

Classifiers include:

  • Destructive command detection (rm -rf, chmod, etc.)
  • Network exfiltration detection (curl to unknown hosts)
  • Privilege escalation detection (sudo patterns)
  • Supply chain attack detection (npm install from URLs)

Denial Tracking

Auto-mode tracks denials per tool and falls back to prompting after threshold:

export const DENIAL_LIMITS = {
  BashTool: 3,
  FileEditTool: 2,
  FileWriteTool: 2,
  default: 3,
}

export function shouldFallbackToPrompting(
  state: DenialTrackingState,
  toolName: string,
): boolean {
  const limit = DENIAL_LIMITS[toolName] || DENIAL_LIMITS.default
  return (state.toolDenials.get(toolName) || 0) >= limit
}

export const AppStoreContext = React.createContext<AppStateStore | null>(null)

export function useAppState<T>(selector: (state: AppState) => T): T {
  const store = useContext(AppStoreContext)
  const selectorRef = React.useRef(selector)
  const storeRef = React.useRef(store)

  const get = React.useCallback(() => {
    return selectorRef.current(storeRef.current.getState())
  }, [])

  return useSyncExternalStore(store.subscribe, get, get)
}

export function useSetAppState() {
  return useAppStore().setState
}

9. Bridge / Remote Control

Bridge Architecture

The bridge enables remote session management via claude.ai:

  • Worker Registration - Daemon registers as worker with session ingress
  • Poll Loop - Polls for work assignments
  • Session Spawning - Spawns claude subprocesses for each work item
  • Heartbeat - Maintains lease on work items
  • Token Refresh - Proactively refreshes JWTs before expiry

10. Query Loop Deep Dive

Query Loop Iterations

The query loop continues until completion or error:

  1. Build Config - Gather env vars, feature flags, model settings
  2. Normalize Messages - Convert internal messages to API format
  3. API Request - Send to provider with streaming
  4. Stream Handling - Process text/tool_use blocks as they arrive
  5. Tool Detection - Identify tool calls in response
  6. Tool Execution - Run tools with permission checks
  7. Result Processing - Convert results to API format
  8. Continue Check - Determine if another iteration is needed

Token Budget

// src/query/tokenBudget.ts
export function checkTokenBudget(
  state: TokenBudgetState,
  currentTokens: number,
): { shouldContinue: boolean; reason?: string } {
  if (currentTokens > state.budget) {
    return { shouldContinue: false, reason: 'budget_exceeded' }
  }
  return { shouldContinue: true }
}

Context Compaction

When context exceeds limits, reactive compaction summarizes old messages:

  • Trigger - Context tokens exceed threshold
  • Pre-compact - Run pre_compact hooks
  • Summarize - Generate summary of old messages
  • Replace - Replace summarized messages with summary
  • Post-compact - Run post_compact hooks

Hook Execution

// src/utils/hooks.ts
export async function executePreToolUseHooks(
  toolName: string,
  input: Record<string, unknown>,
  context: HookContext,
): Promise<HookResult> {
  const results = []
  for (const hook of enabledHooks) {
    if (hook.matches(toolName, input)) {
      results.push(await hook.execute(toolName, input, context))
    }
  }
  return combineHookResults(results)
}
